# Agent Instructions

## Rules

- When creating or updating skills in this repository, use `skills/<skill-name>/SKILL.md`; only create in other directories when the user asks for it.
- When modifying skills, check `CONVENTIONS.md` first and follow it as the repository standard.
- New skills must be **standalone**: self-contained under their package, no runtime dependency on other skills; duplicate shared concepts locally until modularity is supported.
