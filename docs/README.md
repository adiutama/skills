# Shared docs

Use this folder for references and templates shared by multiple skills.

## What goes here

- Cross-skill templates and checklists.
- Shared output schemas and terminology guides.
- Reusable examples that should not be duplicated per skill package.

## What does not go here

- Skill-specific runtime guidance (keep in `skills/<skill-name>/references/`).
- Skill-specific examples/templates (keep in `skills/<skill-name>/assets/`).

## Suggested structure

```text
docs/
├── README.md
├── templates/
└── references/
```
